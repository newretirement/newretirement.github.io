Widget themes

NewRetirement widgets support themes through the use of CSS variables. Designers can supply color values and other substitutions to override the NewRetirement defaults.

To override a CSS variable in a web page, you can create a new style rule that sets the value of the variable to the desired value. This new style rule should be added after the original declaration of the variable, so that it takes precedence.

Here’s a very simple example of how this can be done:

/* Original declaration of the variable */
:root {
  --primary-color: red;
}

/* New style rule that overrides the original value of the variable */
:root {
  --primary-color: blue;
}

In this example, the --primary-color variable is initially declared with a value of red. However, the second style rule overrides this value and sets it to blue instead. This means that any elements on the web page that use the --primary-color variable will now be styled with the color blue.

You can adopt this approach when you embed the widget code by including your own CSS rules after the widget:

<head>
    <!-- ...other HTML... -->
    <script type="module" src="https://widgets-staging.newretirement.com/retirement-calculator/index.js"></script>
    <link href="./your-css-file.css" rel="stylesheet">
    <!-- ...other HTML... -->
</head>

It’s important to note that the new style rule must be added in a way that allows it to take precedence over the existing rule. Broadly speaking, you can do this in two ways:

  1. Ensure the new rule overrides the original rule by placing it after. The example above illustrates this technique.

  2. Ensure the new rule has greater specificity by applying it to a more specific CSS context. We’ll demonstrate this technique below.

How it works​

We’ve included the two widgets above in this documentation page by using the following HTML snippets:

<newretirement-retirement-calculator variant="full" apiKey="AziMr2RAr06LOhAkae2IS4rEZTWErkWu3K5clxAp" url="./rcwidget" env="dev"></newretirement-retirement-calculator>
<div className="alternate-theme">
    <newretirement-retirement-calculator variant="full" apiKey="AziMr2RAr06LOhAkae2IS4rEZTWErkWu3K5clxAp" url="./rcwidget" env="dev"></newretirement-retirement-calculator>
</div>

Then, in the relevant CSS, we’ve added the new style rules to a more specific selector — namely, the “alternate theme” <div>:

:root .alternate-theme {
  --font-family-base: "Open Sans";
  --color-background-paper: #f5f5f5;
  --color-text-secondary: #121212;
  --color-primary-main: #ce1126;
  --color-secondary-main: #157c85;
  --color-accent-main: #157c85;
  --color-divider: #e3e3e3;
  --color-text-tertiary: #ce1126;
  --color-tertiary-400: #eb0029;
  --color-tertiary-500: #c80023;
  --color-tertiary-contrastText: white;
}

Behind the scenes, the widget’s own CSS refers to these variables. If the variables are missing, the widget knows to use the NewRetirement defaults. However, if they’re present, the browser’s CSS engine will apply these variable definitions while laying out the page.

This approach allows us to style the same widget two different ways, even though it appears twice on the same page. The widget code remains the same in each case; only the CSS variables are different.

Using theme variables​

You can use the same variable definitions in a standalone file, or inline within a <style> tag in your HTML document’s <head>. Below we’ve included a bare-bones HTML document that embeds the widget and styles it with the alternate theme:

<!DOCTYPE html>
<html lang="en">
    <head>
        <meta charset="utf-8" />
        <title>Retirement Calculator Web Component</title>
        <link rel="stylesheet" href="https://fonts.googleapis.com/css2?family=Open+Sans:wght@400;500;600&display=swap"/>
        <script type="module" src="https://widgets-staging.newretirement.com/retirement-calculator/index.js"></script>
        <style>
            :root {
                --font-family-base: "Open Sans";
                --color-background-paper: #f5f5f5;
                --color-text-secondary: #121212;
                --color-primary-main: #ce1126;
                --color-secondary-main: #157c85;
                --color-accent-main: #157c85;
                --color-divider: #e3e3e3;
                --color-text-tertiary: #ce1126;
                --color-tertiary-400: #eb0029;
                --color-tertiary-500: #c80023;
                --color-tertiary-contrastText: white;
            }
        </style>
    </head>
    <body style="padding-bottom: 200px;">
        <div style="margin: 1em">
            <h2>Widget Version</h2>
            <newretirement-retirement-calculator
                variant="widget"
                apiKey="AziMr2RAr06LOhAkae2IS4rEZTWErkWu3K5clxAp"
                env="dev"></newretirement-retirement-calculator>
            <h2>Full Version</h2>
            <newretirement-retirement-calculator
                variant="full"
                apiKey="AziMr2RAr06LOhAkae2IS4rEZTWErkWu3K5clxAp"
                signupUrl="https://www.newretirement.com/retirement/src-signup/"
                env="dev"></newretirement-retirement-calculator>
        </div>
    </body>
</html>
